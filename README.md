## How to Determine What Files Are Needed

### Step 1: Ask the User
```
1. Who are you writing FOR? (target audience)
2. What is the goal? (teach, challenge, story, lead magnet)
3. What's your context? (saas founder, coach, freelancer, etc.)
```

### Step 2: Map Audience to Files

| Target Audience | File(s) to Load |
|----------------|-----------------|
| SaaS founders, PMs, developers | `saas-tech.md` + `psychology.md` |
| Marketers, copywriters | `marketing-copywriting.md` + `psychology.md` |
| Coaches, personal dev creators | `coaching-personal-dev.md` + `psychology.md` |
| Freelancers, agency owners | `agency-freelancing.md` + `psychology.md` |

### Step 3: When to Combine Multiple Files

**Real scenarios:**

| Scenario | Files to Load | Why |
|----------|---------------|-----|
| SaaS founder posting about product | `saas-tech.md` + `psychology.md` | Product content for tech audience |
| SaaS founder posting to find customers | `saas-tech.md` + `marketing-copywriting.md` + `psychology.md` | Need both tech credibility + customer messaging |
| Coach wanting clients | `coaching-personal-dev.md` + `psychology.md` | Attract an audience |
| Coach building agency | `coaching-personal-dev.md` + `agency-freelancing.md` + `psychology.md` | Service + business content |
| Freelancer in tech space | `agency-freelancing.md` + `saas-tech.md` + `psychology.md` | Work type + industry combine |
| Marketing agency owner | `marketing-copywriting.md` + `agency-freelancing.md` + `psychology.md` | Expertise + business content |

## Key Principles

1. **Think first** — Don't write until you know which files are needed
2. **psychology is base** — Always load it
3. **Match audience** — Use file matching who you're writing FOR
4. **Combine when needed** — Real posts often need 2+ files
5. **Don't sound like marketing** — If readers feel "selling" → they scroll
6. **Specific > Generic** — Real examples, real numbers, real stories
